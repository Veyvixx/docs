## Initial Setup

Once Nana is in your server, run these commands to get started:

<Steps>
  <Step title="Check that Nana is working">
    Use `/ping` to confirm Nana is online and responsive.
  </Step>
  <Step title="Set up your mod log">
    Use `/modlog <channel>` to designate a channel where moderation actions are posted. This keeps your team informed on every action taken.
  </Step>
  <Step title="Configure your log channel">
    Use `/set log channel <channel>` to enable detailed server event logging — messages, member joins/leaves, role changes, and more.
  </Step>
  <Step title="Set up greet and leave messages">
    Use `/set greet channel` and `/set greet message` to welcome new members automatically. Use `/test greet` to preview it before going live.
  </Step>
  <Step title="Enable Anti-Nuke protection">
    Use `/antinuke status` to open the protection dashboard and configure thresholds to guard your server against raids.
  </Step>
</Steps>

## Recommended Settings

Here are some commands worth setting up early:

| Command | What it does |
|---|---|
| `/modlog <channel>` | Channel where mod actions (ban, kick, warn, etc.) are logged |
| `/set log channel` | Channel for all server event logs |
| `/set greet channel` | Channel for welcome messages |
| `/set jail channel` | Dedicated channel for jailed members |
| `/autorole set` | Automatically assign a role when someone joins |
| `/antinuke toggle` | Enable anti-nuke protection |
